`char[]` (문자 배열)과 `string` (문자열)은 C#에서 서로 다릅니다. `char[]`은 문자의 배열로, 각각의 인덱스에 하나의 문자(`char`)가 저장되며, `string`은 문자들의 시퀀스를 나타내는 불변(immutable) 타입입니다.

### 주요 차이점
- 변경 가능성: `char[]`는 변경 가능합니다. 즉, 배열의 특정 인덱스에 있는 값을 변경할 수 있습니다. 반면에 `string`은 불변으로, 한 번 생성된 문자열의 내용은 변경할 수 없습니다.
- 메서드와 속성: `string` 타입은 텍스트를 조작하기 위한 다양한 메서드와 속성을 가지고 있습니다(예: `Length`, `Substring`, `IndexOf`, `Replace` 등). `char[]` 배열은 이러한 문자열 특화 기능을 내장하고 있지 않습니다.
- 성능: 때때로 `char[]`를 사용하면 많은 문자열 변경 작업을 수행할 때 성능상 이점을 얻을 수 있습니다. 모든 변경 사항에 대해 새로운 `string` 객체를 생성하지 않기 때문입니다.
- 용도: `char[]`는 문자 데이터를 순차적으로 다루거나 크기가 변할 수 있는 시나리오에 유용합니다. `string`은 텍스트를 다루고 읽기 전용 데이터로 작업할 때 더 적합합니다.

### 상호변환
C#에서 `char[]` 배열을 `string`으로 변환하거나 그 반대로 변환하는 것은 매우 쉽습니다. 예를 들어,
* `char[]` 배열을 `string`으로 변환하기 위해서는 `new string(charArray)`를 사용할 수 있고,
* `string`을 `char[]`로 변환하기 위해서는 `stringVar.ToCharArray()`를 사용할 수 있습니다.
